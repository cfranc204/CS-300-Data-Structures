# CS 300 – Data Structures and Algorithms

## Overview

In this course, I worked on implementing and analyzing different data structures to solve practical programming problems. The main focus was understanding runtime efficiency and choosing the right structure for the job.

---

## What problem was being solved?

The main problem in these projects was designing a system that could efficiently store, search, and display course information for an academic advising scenario. The challenge was not just making it work, but making sure it worked efficiently as the dataset grows.

---

## How did you approach the problem?

I approached the problem by first comparing different data structures and thinking about how each one performs. Understanding Big-O notation helped me evaluate trade-offs between speed and memory usage. Instead of jumping straight into coding, I focused on selecting the most appropriate structure for the requirements.

---

## How did you overcome roadblocks?

One of the biggest challenges was handling file input correctly and troubleshooting formatting issues. I also had to debug errors related to file paths and encoding. I overcame these by testing step-by-step, checking outputs carefully, and adjusting the implementation until everything worked properly.

---

## How has this project expanded your approach to designing software?

This project helped me think more about efficiency before writing code. I now consider how data will be stored and accessed before deciding how to structure a program. It reinforced the importance of planning before implementation.

---

## How has this project changed how you write programs?

I now focus more on writing clean, modular code with clear functions and meaningful variable names. I also pay closer attention to error handling and input validation. Overall, this project helped me write programs that are easier to read, maintain, and extend.
